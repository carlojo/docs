## Como crear una llave

Las llaves reguladas se mapearán a los firmantes de Transfiya en una relación uno a uno.

Actualmente, las billeteras solo sirven para conectar múltiples firmantes a un único alias. Dado que estamos haciendo la transición a un modelo donde cada firmante puede tener solo un alias, las billeteras se vuelven redundantes.

<Note>
    Se van a usar los mismos interfaces para crear y actualizar llaves actuales de transfiya.
</Note>

## Flujo de creación de llave

Esto requiere una modificación menor en el proceso de incorporación: necesitamos eliminar el paso de creación de billetera al registrar nuevos usuarios. El flujo de incorporación actual está estructurado de la siguiente manera:

```mermaid
sequenceDiagram
	autonumber
	participant b as Bank
	participant ty as TransfiYa

  b ->> b: Generate keeper (pubKey, privKey)
  b ->> +ty: Create signer (register pubKey)
  ty -->> -b: Signer response
  b ->> +ty: Get wallet ($57...)
  ty -->> -b: Return wallet
  alt wallet exists
    b ->> +ty: Update existing wallet with signer
    ty ->> ty: Update wallet
    ty -->> -b: Return updated wallet
  else wallet doesn't exist
    b ->> +ty: Create a new wallet with signer
    ty ->> ty: Create wallet
    ty -->> -b: Return wallet
  end
  ```
