## Sobre cuentas

Las cuentas bancarias en Transfiya se manejan como otro tipo de llave o signer en el sistema.

<Note>
    Hoy en día Transfiya permite enviar transferencias directas a llave (signer) sin aceptación de la cuenta destino. Esto significa que el usuario que recibe la transferencia no necesita confirmarla antes de que se realice la transferencia.
</Note>

## Transferencias cuenta a cuenta

Para enviar transferencias cuenta a cuenta, es necesario especificar el destino de la transferencia. La plataforma soporta el modelo de envío directamente a credencial de pago (signer) o envío a un identificador de cuenta bancaria.

### Envío a credencial de pago (signer) conocido

En caso de que se conozca la credencial de pago o signer destino, la transferencia se ejecuta en tiempo real sin proceso de aceptación.

```Javascript
"source": "wXxwpxB32saqfmfMxAQD4SVWWhhn6akLC2",
"target": "wfvmRir8XxRtoEuUBDkv3BP4T7BQezwm1f"
```

### Envío a cuenta no registrada en sistema

El identificador de cuenta bancaria representa el tipo de cuenta, el número de cuenta y el banco de destino. Este identificador se utiliza para enviar transferencias a destinatarios que requieren un proceso de verificación o aceptación.

Para enviar a estas cuentas se usa el mismo modelo de crear transferencias en el sistema Transfiya usando numero de celular.

```Javascript
"source": "wXxwpxB32saqfmfMxAQD4SVWWhhn6akLC2",
"target": "svgs:42424242@bancoazul.co"
```

El modelo de identificador de cuenta bancaria es el siguiente:

(tipo de cuenta):(número de cuenta)@(dominio de banco destino)

<Note>
    Este modelo permite enviar transferencias a cuentas bancarias no registradas en el sistema Transfiya e incluso a otras redes dentro o fuera del país.
</Note>
