## Modelo Transfer

### Concepto POST /transfers

Transfiya utiliza el concepto de "transfer" para iniciar pagos en el sistema.
El *Transfer* es un inciacion de pago compuesta de transacciones que representan dos o más movimientos de dinero entre los participantes.

Este modelo es agnóstico al caso de uso y soporta cualquier tipo de movimiento de fondos, dependiendo de las reglas configuradas del sistema y del nivel de origen y destino del envío.

### Origen y destino de transferencia

Plataforma de Transfiya permite envios entre diferentes llaves en el sistema desde su inicio. Principal uso de origen y destino es simplificar el proceso de envio y hacerlo agnostico a caso de uso.




<Note> El modelo de aceptación simplifica significativamente el proceso de incorporación de clientes, ya que elimina la necesidad de crear una llave de manera previa.</Note>

<Tip> Paso de aceptacion no es necesario para procesar el pago con destino conocido.</Tip>

### Flujos de dinero

Cuando se inicia un transfer se activa un flujo de pago.

Cada flujo de pago inicia una máquina de estados que se encarga de gestionar la operación entre participantes y sistemas externos. Maquina de estados se define durante inciacion de transfer usando campo transfer.labeles.type:

<ResponseField name="labels.type" type="string[SEND, SENDMOL, REQUEST, ONBOARDING]">
    Tipo de transferencia.
</ResponseField>

La máquina de estados permite controlar y orquestar el flujo mediante pasos específicos que dependen de la definición de los pasos del flujo, el entorno y el tipo de participantes representados por el firmante (signer).

<Note> El uso del modelo de transfer para iniciar pagos en el sistema permite emplear el mismo protocolo y REST API para cualquier caso de uso o flujo de dinero, incluyendo el nuevo flujo regulatorio. </Note>

## Crear Transferencias regulatorias (SENDMOL o push mol)

Todos los movimientos de dinero utilizan el mismo concepto de transferencia para iniciar los flujos.

Para identificar las transferencias que utilizan el modelo de liquidación con MOL, el sistema emplea un flujo específico SENDMOL.

<Note> La única diferencia entre el flujo de envío estándar y el flujo regulatorio radica en que este último requiere una autorización adicional del banco central para completar el proceso de liquidación. </Note>

El otro punto de flujo regulatorio propuesto exige la resolución de la llave antes de iniciar la transferencia.

> Esto implica que dicho flujo no permite iniciar transferencias con participantes que no tengan una llave previamente registrada en el sistema.

Es importante destacar que la mayoría de las entidades financieras replicarán sus llaves en varios sistemas SPI. Como consecuencia, la mayor parte de las transferencias operarán bajo un modelo intersistema o dentro del propio ecosistema de Transfiya.
