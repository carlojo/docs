El flujo de dinero en tiempo real de personas a empresa moderniza muchas de las operaciones tradicionales de este tipo, sin embargo el potencial innovador es equiparable o incluso mayor a los casos empresa a persona. Los pagos en tiempo real son el principal foco, permitiendo a las empresas tener liquides inmediata con cada transacción por cada pago que reciben de sus clientes, una dinámica que durante mucho tiempo a sido exclusiva del efectivo o transacciones intra-bancarias. Ejemplos de casos de uso de pagos a empresas van desde las compras presenciales o en linea, como también recarga de cuentas virtuales (apuesta, videojuegos, puntos, etc), pagos de productos de créditos y servicios públicos.

ACH también se apalanca en su funcionalidad de **request-to-pay** que permite generar solicitudes de pago donde el usuario-persona recibe una notificación de solicitud de pago, entra a la app de su banco y aprueba el pago, creando una experiencia de usuario rápida y transparente. Las entidades pueden habilitar a sus clientes empresas para generar solicitudes de pago individuales o de forma masiva a través de API o incluso su portal empresarial. Algunos de los ejemplos son solicitudes periódicas para pagos de suscripciones, solicitud de pago de servicios públicos o telcom, recarga de cuentas digitales (trading, millas, videojuegos, apuestas, etc)

A continuación encontrarás una lista de formas en las cuales se podrán realizar transacciones **personas > empresa** al igual que los casos de uso en los que aplica.

### Carga de solicitudes de pago masivas

Este formato funciona igual igual que la carga de un archivo para enviar dinero, con la diferencia que crea solicitudes de pago. Esto quiere decir que las empresas pueden enviar un archivo via API o ingresar al portal empresarial de sus entidades o de ACH para montar sus archivos y realizar cobros a sus pagadores, los cuales tendrán que aceptarlo en sus apps para que se complete la transferencia, a pesar que la carga sea masiva la empresa podrá tener tracking individual de cada una de las solicitudes de pago.

| Canal | Casos de uso |
| --- | --- |
| Archivos en portal empresarial de la entidad o portal ACH | - Cobros de recaudo masivos |
| Archivos via API | - Cobros automatizados de servicios públicos
- Cobros automatizados de servicios bajo suscripciones (prepago)
- Cobros automatizados de productos de crédito
- Cobros automatizados de impuestos y parafiscales |

### Pagos 1a1 por demanda

En los pagos 1a1 las empresas a través de una conexión por API podrán iniciar pagos en aplicaciones propias o de terceros para recibir pagos, esto apalancándose en diferentes experiencias para iniciar el pago en la punta persona como QR o request to pay.

| Canal | Casos de uso |
| --- | --- |
| API request to pay | - Recarga de cuentas en otras entidades (Top-ups, consignaciones)
- Recarga de cuentas digitales (Trading, millas, apuestas, etc)
- Pago de suscripciones (Prepago)
- Pagos recurrentes (Seguros, educación, transporte, etc)
- Pagos en puntos de venta iniciados en el POS |
| API QR | - Pagos presenciales
- Pagos en linea |
| Solicitud individual portal o app del banco | - Cobros de emergencia |

### PSE en linea | X2B

Actualmente PSE realiza el débito en linea a un usuario y el crédito a la empresa en batch. Nuestra nueva propuesta lleva el crédito al comercio al instante de tal forma que las empresas puedan acceder a sus dinero al instante.

### Botón TY (PSE.ya)

El botón transfiya se enfoca en ofrecer una solución de pagos en linea que se apalanca en las llaves utilizando el modelo request to pay. Al realizar un pago, los usuarios recibirán una notificación en la app de su banco con el valor, el cual tendrán que aceptar o rechazar, ofreciendo una experiencia mas limpia y satisfactoria.
